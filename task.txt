Создать таблицы projects и servers. В качестве СУБД, например, postgres, mysql(mariadb). 
В одном проекте может быть несколько серверов, но не больше 3ёх,
но один сервер не может принадлежать сразу нескольким проектам, только одному -
 по реализации связи между таблицами подумать.
В каждой таблице обязательные колонки id, name.
Значение колонки id генерируется автоматически при добавлении, колонка является первичным ключом.
Значения колонки name - нельзя заносить пустые и null значения. Ограничение по символам - 20,
 не допускаются числовые символы.
Предоставить скрипты создания таблиц.
 
Реализовать REST API эндпоинты для следующих операций:
- добавление записи в таблицу projects
- подгрузка всех записей из таблицы projects
- добавление записи в таблицу servers
- подгрузка всех записей из таблицы servers
- обновление записи в таблице servers по ее id
- удаление записи из таблицы servers
Если операция содержит ответ - использовать формат json.
Нельзя добавлять сервера без проектов.
 
Обязательно написать unit-тесты - покрыть функционал взаимодействия с БД предусмотрев все вышеуказанные ограничения.
 Использовать модуль unittest. Если понадобится, то подключить mock.
 
В качестве orm использовать sqlalchemy. Для реализации rest взаимодействия использовать flask.
